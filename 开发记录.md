## Codemagic 配置要点

### codemagic.yaml 构建流程
1. `flutter create` — 在云端生成 iOS 项目文件
2. `flutter pub get` — 获取依赖
3. `xcode-project use-profiles` — 自动配置代码签名
4. `xcode-project build-ipa` — 构建 IPA 文件

### 证书上传位置
Settings → Code signing identities
- iOS 证书：上传 `.p12` 文件 + 密码
- iOS 配置文件：上传 `.mobileprovision` 文件

### 构建结果
- IPA 文件通过邮件发送到 3371574658@qq.com
- 免费额度：每月 500 分钟 macOS 构建时间

## 踩坑记录

### 1. Git 身份未配置
```
git config user.name "shendehao"
git config user.email "shendehao@users.noreply.github.com"
```

### 2. Bundle ID 不匹配
- Codemagic 截图显示 `com.durian711.apple315`
- 实际 Profile 中是 `app.durian7111.apple315`
- 通过解析 mobileprovision 文件中的 `application-identifier` 字段确认

### 3. export_options.plist 不存在
- 初始配置引用了不存在的 `/Users/builder/export_options.plist`
- 改用 Codemagic 原生工具 `xcode-project use-profiles` + `xcode-project build-ipa` 解决

### 4. iOS 目录缺失
- 项目在 Windows 上创建，没有 `ios/` 目录
- 在 codemagic.yaml 中添加 `flutter create` 步骤在云端生成

### 5. 证书文件误提交 Git
- `.p12` 和 `.mobileprovision` 被提交到 GitHub
- 通过 `git rm --cached` 移除并更新 `.gitignore`
